Apple Health Processing Challenges

Reconciling irregular, sparse, and asynchronous physiological data

Irregular Sampling

Values are not recorded uniformly. Heart rate may appear multiple times per minute, while sleep stages span hours.

Required careful timestamp normalization to minute boundaries.

Sparse Metrics

Energy metrics update only during specific activities. VO₂ Max appears infrequently.

Implemented forward-fill and minimal interpolation strategies.

Sleep Intervals

Sleep exists as intervals, not per-minute samples. Stages transition at irregular times.

Required interval-to-minute conversion logic.

Workout Redundancy

GPX workout data may provide redundant timestamps that conflict with XML records.

Deduplication and priority rules established.

Apple Health Processing Pipeline

From raw XML to minute-synchronized Pandas DataFrame

  • Parse XML structure and extract each metric into a Pandas table
  • Normalize all timestamps to the minute boundary (floor operation)
  • Merge all metrics into a single table indexed by minute
  • Apply forward-fill for missing values where appropriate
  • Ensure consistent alignment across all 36 metrics for downstream modeling
  • Validate data integrity through statistical profiling and visual inspection

Result: A comprehensive minute-wise Apple Health dataset with complete temporal coverage and no alignment drift

RouteSense Screen-Time Inference

Behavioral modality without OS-level access

The Challenge

Traditional screen-time tracking requires OS-level permissions and integration. This creates privacy concerns and platform limitations.

RouteSense Solution: Infer screen activity from network handshake patterns—an indirect but powerful method.

The Method

When users actively interact with applications (messaging, browsing, social media, streaming), devices generate outbound packet bursts.

Key Observation: Passive background activity is low-volume. Active interaction correlates with >1MB/min network bursts.

09

RouteSense Detection Algorithm

Transforming network patterns into behavioral signals

Binary Classification Heuristic

screen_on(t) = 1 if upload_byte_count(t) > 1000 KB

This creates a minute-level binary time series representing behavioral engagement intensity.

Advantages

  • OS-agnostic and privacy-preserving
  • Fine temporal resolution (per minute)
  • Reflects real behavioral intensity
  • Captures WhatsApp, browsing, content consumption

Limitations & Validation

  • Network bursts may occur without explicit screen use
  • Some apps use background data
  • Threshold requires empirical validation
  • Cross-validated against manual screen logs

LSTM Model Architecture

Predicting next-minute average heart rate

Objective

Predict the next-minute average heart rate using all fused multimodal and engineered features. This captures micro-trends in physiological response driven by behavior, environment, and recent activity.

Model Philosophy

The LSTM architecture serves as a nonlinear pattern recognizer, temporal context synthesizer, and adaptive estimator for short-term cardiovascular trends. It learns to associate behavioral events (screen bursts) and environmental shifts (temperature changes) with immediate HR responses.

17

LSTM Feature Expansion

Enriching the input space before modeling

  • Polynomial Features: Squared and cubic transformations to capture non-linear relationships
  • Noise Augmentation: Gaussian noise-perturbed replicas to improve robustness
  • Rolling Statistics: Mean and variance of heart rate over multiple time windows
  • Interaction Terms: Cross-products between screen activity, weather, and physiological variables
  • Normalization: Z-score standardization across all features for stable training

This expansion increases model expressiveness without requiring explicit deep time windows, allowing the network to learn complex patterns from enriched single-timestep inputs

Hybrid CNN + Transformer + LSTM

Resting heart rate baseline prediction

Motivation

Resting heart rate (RHR) is a powerful marker of stress, fatigue, recovery, sleep debt, and autonomic balance.

Unlike instantaneous HR, RHR reflects longer-term physiological state and adaptation to lifestyle patterns.

Architecture Philosophy

Predicting RHR from multimodal context requires understanding how behavior and environment shape physiological baseline.

We combine three complementary representation systems for maximum expressiveness.

22

Hybrid Model Architecture

Three parallel representation systems

CNN Branch

Purpose: Structural feature extraction

Convolutional layers detect local patterns and spatial relationships within the feature vector, identifying co-occurring physiological and behavioral signatures.

Transformer Branch

Purpose: Feature-wise relational attention

Self-attention mechanisms capture dependencies between distant features, learning which variables influence RHR across the entire feature space.

LSTM Branch

Purpose: Implicit temporal structure

Recurrent layers model temporal dependencies and sequential patterns, even with single-step sequences through learned state representations.

These three branches process the same input in parallel, each contributing unique perspectives on the data's structure

Hybrid Model Training

Optimization for baseline physiological prediction

Training Strategy

Unlike the LSTM model, the hybrid architecture is trained with shuffled samples rather than strict chronological ordering.

This approach helps the model generalize across diverse contexts and prevents overfitting to temporal artifacts.

Generalization Focus

The model learns to predict resting HR from contextual patterns rather than memorizing sequences.

This makes it robust to variations in daily routines and behavioral patterns.

INTRODUCING

EXO-Hypermind
Framework

Quantifying the predictability of human physiology

The Central Question

"How predictable is your physiology under the interplay of behavior and environment?"

Rather than predicting physiology directly, EXO-Hypermind evaluates system stability, irregularity, model surprise, and context-driven fluctuations

27

EXO-Hypermind Philosophy

From prediction to stability assessment

Traditional Approach

Conventional models predict future values: "What will your heart rate be?"

This provides point estimates but limited interpretability about overall physiological coherence.

EXO-Hypermind Approach

Instead asks: "How stable and predictable is your physiological system?"

Transforms prediction error into a stability index that quantifies system coherence and irregularity.

This shift enables context-aware health readiness metrics applicable across digital health, wellness analytics, and performance monitoring

EXO-Hypermind Composites

Condensing multimodal data into interpretable indices

Screen Composite

Behavioral Intensity Score (0-100)

Aggregates screen-on duration, upload volume, and engagement streaks into a single digital behavior metric.

Weather Composite

All three composites are scaled 0-100 for intuitive interpretation and direct comparability

EMA Smoothing Layers

Revealing trends across multiple timescales

Fast EMA (30 minutes)

Captures immediate stability shifts responding to acute stressors, sudden behavioral changes, or environmental transitions.

Sensitive to real-time fluctuations, useful for alerting to rapid destabilization.

Slow EMA (4 hours)

Reveals longer-term trends such as gradual sleep debt accumulation, sustained stress periods, or recovery trajectories.

Filters out transient noise, highlighting persistent patterns in physiological stability.

The dual-EMA approach allows users to distinguish between temporary disruptions and sustained stability shifts
